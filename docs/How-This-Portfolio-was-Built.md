## How This Portfolio was Built

This document explains the approach, tools, and workflow used to create this software technical writing portfolio.

## Purpose of This Portfolio
The goal of this portfolio is to demonstrate software technical writing skills using a docs-as-code approach. The focus is on clarity, task-based documentation, and industry-standard tooling rather than on a specific software product.

## Documentation Approach
The documents in this portfolio follow a task-oriented approach:
-   Each document focuses on a single user goal
-   Steps are written clearly and sequentially
-   Prerequisites and notes are included where necessary
-   The style is intentionally simple and user-focused.

## Tools Used
The following tools were used to create this portfolio:
1. Identify a common user track
2. Observe the application behaviour
3. Write the procedure in Markdown
4. Review content for clarity and consistency
5. Commit changes using Git
6. Publish updated to GitHub

## Target Audience
This portfolio is intended for:
- Hiring Managers
- Technical Writing Teams
- Software documentation stakeholders
